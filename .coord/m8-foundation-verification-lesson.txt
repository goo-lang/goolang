M8 Foundation Verification shipped 12 of the targeted ~18 basic constructs after surfacing five layered foundation bugs that had been masking each other. (1) Parser: the gpu_intrinsic/sync/alloc/copy rules in primary_expr matched any ident.ident shape and hijacked package method calls (fmt.Println parsed as a CUDA intrinsic); kernel_launch had the further problem of `<<<` lookahead colliding with `i < N` comparisons, so every if/for using `<` failed to parse. (2) Parser: func_signature's "LPAREN func_params RPAREN func_result" alternative had a year-old TODO that returned only the params and dropped the result, leaving every function with both params and a return type registered as `() -> void` — every call site failed type comparison with "Cannot compare incompatible types void and int32". (3) Codegen: function_codegen's parameter loop checked `param->type == AST_IDENTIFIER` but the parser builds params as AST_VAR_DECL, so the loop body never ran and every parameter referenced in a function body resolved to "Undefined identifier" at codegen. (4) Codegen: LLVMGetElementType doesn't work with LLVM 22 opaque pointers — used for both function calls (build), variable loads (`load ptr, ptr %x` produced `ptr` instead of the underlying integer), and the println call. Each had to be rewritten to use LLVMGlobalGetValueType for functions and explicit codegen_type_to_llvm(goo_type) for loads. (5) Architectural: the type-checker scope is gone by codegen time, but codegen still calls type_check_binary_expr / type_check_call_expr for type info, which trigger identifier lookups. Function params and var decls now mirror into the type-checker scope at codegen time so those lookups resolve. Also: codegen_get_basic_type was missing TYPE_STRING, causing LLVMGetUndef(NULL) SIGSEGVs on any string literal; type_check_function_decl's `// TODO: proper signature` was a year old and load-bearing. The 8 deferred constructs (struct, slice-literal, error-unions, for-range, multi-return, nullable, map-literal, postfix-incr) are each multi-layer plumbing projects, not one-rule grammar additions, and are tracked as M8-* child tasks ordered by leverage. The gate as drafted (15 constructs) turned out to be unreachable in a single /goal session because of the layered-bug discovery pattern — each fix unmasked the next; M8 closes with the foundation done and the remaining constructs staged for follow-on work.
