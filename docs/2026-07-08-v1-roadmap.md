# Goo v1 Roadmap — evidence-based synthesis (refresh)

**Date:** 2026-07-08  
**Source:** `goo-v1-roadmap` workflow (run wf_4fc64c40-b4f) — 17 agents, ~1.25M tokens, every claim checked by running `bin/goo` (branch feat/spmd-harness at PR #162, post interfaces/RTTI/generics/arenas/comptime/SPMD).  
**Supersedes:** `docs/2026-06-30-v1-roadmap.md` as the working v1 plan.

## v1 Definition

> v1 = bin/goo reliably compiles AND runs a useful Go-compatible subset (gofmt-formatted ordinary Go: grouped var, switch-init, labels/goto/fallthrough, raw strings, trailing commas, multi-method interfaces, channels with close/range, defer-in-loop, method values, package-scope globals, nil pointers/slices/maps per the decided semantics), with Goo's differentiators — error unions !T (error/try/catch) and nullables ?T (if-let) — working end-to-end including their interop edges (user (T,error) bridging, e.Error(), ?T==?T), zero known silent miscompiles or signal crashes, a stdlib covering fmt/strings/strconv/os/errors/math/time/sync with type-checked shim signatures and cross-package types/methods, an honest CLI (build/run, correct exit codes, --emit-llvm that does what it says, real -O), and a green data-driven e2e gate (golden run + reject fixtures with timeouts and exit-code assertions) runnable on a clean machine without CompCert. Explicitly NOT v1: self-hosting, GC, GPU/WASM targets, macros, async runtime, capability security, formal verification, package manager, working LSP/debugger/REPL.

## Verified baseline (real state)

VERIFIED real state (2026-07-08, branch feat/spmd-harness): The core pipeline WORKS end-to-end — lexer -> bison parser -> type checker -> LLVM IR (verified in-pipeline for the emit paths that gate) -> gcc link against lib/libgoo_runtime.a -> runnable ELF. 321/321 golden fixtures compile, run, and stdout-match (~15s). Targeted ASI is implemented and correct (the 'no-ASI blocker' claim was REFUTED); linking works (claim it never worked REFUTED); structs, value/pointer-receiver methods, interfaces with dynamic dispatch, type switches, closures, defer, goroutines/channels/select, generics via monomorphization, and comptime monomorphization all work end-to-end (claims of zero support REFUTED; comptime is genuinely wired, unlike the other 'advanced' features). Differentiators !T (error/try/catch, cross-frame propagation) and ?T (if-let, no zero-value sentinel bug) work end-to-end for int/string/float/struct, including strconv.Atoi bridged as !int. HOWEVER: (1) proven silent miscompiles exist — `v, ok := <-ch` emits TWO goo_chan_recv calls causing spurious deadlock aborts, and invalid tokens (TOKEN_UNKNOWN, e.g. `#`) are silently dropped so garbage source compiles; ## Critical path

1. CORRECTION FIRST: the presumed keystone (an ASI lexer fix) was REFUTED by adversarial verification — targeted ASI already works end-to-end (semicolon-free programs compile and run), so no ASI rewrite is on the path. The real keystone is Phase 0 soundness: the compiler currently miscompiles comma-ok channel receives, silently accepts invalid tokens, and SIGILLs on !?T with zero diagnostics — nothing downstream is trustworthy until these and the harness gaps (no timeouts, no exit-code checks) are fixed.
2. P0.1 comma-ok double-recv fix + P0.2 LLVM verify gate + P0.3 !?T rejection + P0.4 TOKEN_UNKNOWN gate — the four soundness fixes; all subsequent phases assume a compiler that fails loudly.
3. P0.7-P0.9 golden-harness hardening (timeouts, exit codes, reject harness) — the regression net every later phase's gate depends on.
4. P1.1 interface grammar fix (void method + another method) — gates P2.1 method-set enforcement and P2.10 union constraints; without it multi-method interfaces are unusable so interface-correctness work is blocked.
5. P2.2 nil-semantics decision — blocks every Go pointer/slice/map-walking program; both branches are ~equal effort but the decision is human-owned and front-loads or unblocks a large slice of Go compatibility.
6. P0.1 -> P3.1 close() -> P3.2 range-over-channel — the channel-lifecycle chain that makes concurrent Go programs expressible.
7. P4.2 qualified type names -> P4.3 cross-package methods — the hard ceiling on all stdlib growth via vendoring; time/sync/Builder-style APIs are impossible until this lands.
8. P4.11 stdlib smoke suite -> Phase 5 release gate (verify-core green on clean machine + docs truth pass) — the measurable definition of shipped.
