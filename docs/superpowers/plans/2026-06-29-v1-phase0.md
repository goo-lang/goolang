# v1 Phase 0 — Usability + Severe-Miscompilation Fixes (Implementation Plan)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Make `bin/goo` usable as a real tool (any cwd) and stop the two ways it silently emits WRONG or INVALID output (break/continue no-ops; invalid IR when printing non-scalars), and stand up a trustworthy end-to-end golden-test oracle.

**Architecture:** Five surgical fixes to the codegen/runtime path, plus a data-driven golden runner. A new binary-relative runtime-path resolver; a loop-context stack on `CodeGenerator` for break/continue; a clean codegen error for unsupported print args; symmetric `.o` cleanup; a bash golden runner wired into `make`.

**Tech Stack:** C23, LLVM C API, the Goo compiler (`bin/goo`), GNU Make, bash.

**Spec:** `docs/2026-06-29-v1-roadmap.md` (Phase 0). This is the correctness floor — nothing downstream is trustworthy until these land.

- [ ] **Step 3: Implement the resolver + use it at all link sites**

In `src/codegen/codegen.c`, add above `codegen_emit_executable` (the function containing the link commands). Include `<unistd.h>` and `<sys/stat.h>` if not present.

```c
// Resolve the runtime archive path independent of cwd so the compiler works
// when invoked from any directory. Order: $GOO_RUNTIME -> <exe-dir>/../lib ->
// cwd-relative fallback. Returns a pointer into a static buffer (single-threaded
// codegen; not reentrant — acceptable here).
static const char* goo_runtime_archive_path(void) {
    static char buf[4096];
    const char* env = getenv("GOO_RUNTIME");
    if (env && env[0] != '\0') { snprintf(buf, sizeof(buf), "%s", env); return buf; }
#ifdef __linux__
    char exe[4096];
    ssize_t n = readlink("/proc/self/exe", exe, sizeof(exe) - 1);
    if (n > 0) {
        exe[n] = '\0';
        char* slash = strrchr(exe, '/');        // dir-of-exe
        if (slash) {
            *slash = '\0';
            snprintf(buf, sizeof(buf), "%s/../lib/libgoo_runtime.a", exe);
            struct stat st;
            if (stat(buf, &st) == 0) return buf; // exists relative to the binary
        }
    }
#endif
    snprintf(buf, sizeof(buf), "lib/libgoo_runtime.a");  // fallback (repo-root cwd)
    return buf;
}
```

Then replace the literal `lib/libgoo_runtime.a` in each link-command `snprintf` (the `__APPLE__`, `__linux__`, generic-Unix, and `_WIN32` branches) with `%s` fed by `goo_runtime_archive_path()`. Example for the Linux branch:

```c
    snprintf(link_command, sizeof(link_command),
             "gcc -no-pie -o %s %s %s -lm -lpthread",
             filename, object_filename, goo_runtime_archive_path());
```

(`bin/goo` lives at `<repo>/bin/goo` and the lib at `<repo>/lib/` — so `<exe-dir>/../lib` resolves correctly for the installed/dev layout.)

Run: `eval $(opam env --switch=default) && make cwd-link-probe`
Expected: `cwd-link-probe: PASS`. Also re-run `make verify` to confirm repo-root linking still works.

**Files:**

- [ ] **Step 3: Implement the loop-context stack**

In `include/codegen.h`, add to `struct CodeGenerator` (after the struct-cache fields):
```c
    // Loop-context stack for break/continue targets (depth-bounded; nesting
    // deeper than 32 is rejected with a codegen error).
    LLVMBasicBlockRef loop_break_bb[32];
    LLVMBasicBlockRef loop_continue_bb[32];
    int loop_depth;
```

In `src/codegen/statement_codegen.c`:
- Add static helpers near the top:
```c
static int codegen_push_loop(CodeGenerator* cg, LLVMBasicBlockRef brk, LLVMBasicBlockRef cont) {
    if (cg->loop_depth >= 32) return 0;       // too deep — caller emits codegen_error
    cg->loop_break_bb[cg->loop_depth] = brk;
    cg->loop_continue_bb[cg->loop_depth] = cont;
    cg->loop_depth++;
    return 1;
}
static void codegen_pop_loop(CodeGenerator* cg) { if (cg->loop_depth > 0) cg->loop_depth--; }
```
- In `codegen_generate_for_stmt`, for BOTH the range path and the C-style path: after the loop's body/cond/exit/post blocks are created and BEFORE generating the body statements, `codegen_push_loop(codegen, <exit_bb>, <continue_target>)` — where `continue_target` is the **post/increment block** for the C-style loop and the **cond block** for the range loop (so `continue` re-runs the increment/next-index). `codegen_pop_loop(codegen)` after the body. (Read the C-style branch below line ~430 to identify its post/increment and exit blocks.)
- Replace the stub:
```c
        case AST_BREAK_STMT:
            if (codegen->loop_depth == 0) { codegen_error(codegen, stmt->pos, "break outside loop"); return 0; }
            LLVMBuildBr(codegen->builder, codegen->loop_break_bb[codegen->loop_depth - 1]);
            // Subsequent statements in this block are unreachable; start a fresh
            // block so later codegen has a valid (dead) insertion point.
            codegen_set_insert_point(codegen, codegen_create_block(codegen, "after.break"));
            return 1;
        case AST_CONTINUE_STMT:
            if (codegen->loop_depth == 0) { codegen_error(codegen, stmt->pos, "continue outside loop"); return 0; }
            LLVMBuildBr(codegen->builder, codegen->loop_continue_bb[codegen->loop_depth - 1]);
            codegen_set_insert_point(codegen, codegen_create_block(codegen, "after.continue"));
            return 1;
```
(`codegen_create_block`/`codegen_set_insert_point` are already used in this file — reuse them.)

Run: `eval $(opam env --switch=default) && make break-probe continue-probe`
Expected: both PASS (`0 1 2` / `0 1 3 4`). Then a nested-loop sanity check: a `break` in an inner loop must exit only the inner loop — add `examples/break_nested.goo` (+ expected) asserting the outer loop continues. ### Task 5 (P0-5): Data-driven golden runner + fold the Phase 0 fixtures in

**Files:**
- Create: `scripts/run_golden.sh`
- Modify: `Makefile` (add `test-golden` target)

**Interfaces:**
- Produces: `scripts/run_golden.sh` — discovers every `examples/*.goo` that has a sibling `examples/*.expected.txt`, compiles each via `$COMPILER` (default `bin/goo`), runs the binary, diffs stdout against the expected file, prints per-case `PASS`/`FAIL`, and exits non-zero iff any case fails.

- [ ] **Step 1: Write the runner**

Create `scripts/run_golden.sh`:
```bash
#!/usr/bin/env bash
# Data-driven golden runner: every examples/<name>.goo with a sibling
# examples/<name>.expected.txt is compiled via bin/goo, run, and its stdout
# diffed against the expected file. Exit non-zero iff any case fails.
set -u
COMPILER="${COMPILER:-bin/goo}"
EX_DIR="${EX_DIR:-examples}"
pass=0; fail=0; failed=()
for goo in "$EX_DIR"/*.goo; do
    exp="${goo%.goo}.expected.txt"
    [ -f "$exp" ] || continue
    base="$(basename "$goo" .goo)"
    out_bin="build/golden_${base}.out"
    mkdir -p build
    if ! "$COMPILER" "$goo" -o "$out_bin" >/dev/null 2>"build/golden_${base}.cerr"; then
        echo "FAIL  $base (compile/link)"; fail=$((fail+1)); failed+=("$base"); continue
    fi
    actual="$("$out_bin" 2>/dev/null)"
    if [ "$actual" = "$(cat "$exp")" ]; then
        echo "PASS  $base"; pass=$((pass+1))
    else
        echo "FAIL  $base (output mismatch)"; diff <(printf '%s' "$(cat "$exp")") <(printf '%s' "$actual") | head -20
        fail=$((fail+1)); failed+=("$base")
    fi
done
echo "--- golden: $pass passed, $fail failed ---"
[ "$fail" -eq 0 ]
```
Make it executable: `chmod +x scripts/run_golden.sh`.

Add the Makefile target:
