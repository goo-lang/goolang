> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `goostd/lanes` ships: `Partition(arr, comptime count)` splits a slice into disjoint per-lane views whose safety the compiler PROVES (move + single-goroutine attribution + view-capture rejects), demonstrated by a 1D heat-equation stencil bit-identical to a serial reference, monomorphized per comptime `count`, gated in `make verify-core`.

**Architecture:** (1) Lift the package-function comptime wall (`type_checker.c:1018-1020`) so a vendored package can declare comptime params; cross-package monomorphization with package-qualified mangling. (2) Pure-Goo `goostd/lanes` on existing runtime primitives — capacity-1 channels for halos, goroutine-per-lane, one backing `[]float64` pre-sliced into disjoint tiles. (3) A NEW narrow checker pass `src/types/lane_ownership.c` (wired from `type_check_program` pass 2) enforces the four proof obligations, reusing `ownership_checker.c`'s bones (`Variable.is_moved`, `mark_variable_moved`) only for Partition-origin values. The escape analyses (`param_escape.c`/`block_escape.c`) are NOT modified — they are codegen-side optimization; the obligations are checker-side rejections that mirror Sink #4's boundary.

**Tech Stack:** C23 compiler sources; Goo for `goostd/lanes` + fixtures; LLVM-C; Makefile probes; helgrind (availability decided by Task 1).
