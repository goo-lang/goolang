## Context to remember
- The `is_keyed` flag on struct literals is currently stashed on `head->node_type` low bit via a `(Type*)(intptr_t)1` kludge — documented in the parser action. The first impl child should refine this into a real AST field (probably an array of `(field_name_or_null, expr)` pairs).
- AST_STRUCT_LITERAL is at the END of the AST node enum, deliberately. Adding to the middle silently shifts every other value and breaks switch dispatch — learned the hard way mid-session.
- The lexer-layer frame stack uses `m10_paren_depth` which counts LPAREN + LBRACKET (not just LPAREN). Bracket depth matters because `[]int{1,2,3}` inside a for cond must keep the FOR frame alive — the slice lit's `{` is at bracket depth 1, not 0.
- COLON pops the topmost IF_COND frame at matching paren_depth. This handles `case Foo if x > 0:` guard conditions where the IF ends in COLON rather than LBRACE. Without this pop, the frame leaks past the case and the case-body's first brace gets wrongly recolored.
- TOKEN_MATCH isn't currently mapped to a Bison token in `lexer_bridge.c` — match isn't lexable yet despite the grammar having `match_expr` rules. MATCH_COND frame is reserved (constant defined, push logic ready) but unused. When TOKEN_MATCH is wired, the push needs to fire too.
- `make verify` is the canonical regression net: baseline-probe + smoke-stdlib + v2-bootstrap-pilot + comptime-block-probe + comptime-probe (5 gates). All green at handoff.
- Unit tests are now runnable: `make test-lexer`, `make test-codegen`, `make test-units`, `make comptime_test`. Useful for focused regression checks.
