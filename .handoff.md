Small ledgered follow-ups (from arc 14/15 reviews plus PR disposition):
- RESOLVED (arc 18): receiver-kind soundness. Gap assessment found the
  checker-level fix (direct + embedded satisfaction) had already landed
  via 8d012cd; the remaining real gap was the closed-world RTTI path —
  codegen_interface_target_match built value-form vtables
  unconditionally, so a value-boxed T wrongly satisfied a
  pointer-receiver interface at runtime. Fixed by gating each candidate
  form on type_interface_satisfied; pinned by
  iface-recv-kind-rtti-probe + iface_target_recvkind_probe.
- Dup-case detection misses stamped float literals: `switch n { case 2:
  case 2.0: }` accepts (first-match-wins, no crash — expr switches lower to
  a compare chain) where Go rejects "duplicate case". Mandate A's fold is
  int-shape-gated; a post-stamp fold attempt would close it.
- The one-token-lookahead position offset (diagnostics land on the next
  token's line) is confirmed to affect literal and case-clause positions,
  not just statement positions — same parser-owned root cause as item 1.
- Pre-existing double-diagnostic on type-switch vtable failure (specific +
  generic "internal: cannot build type switch vtable compare" both fire).

- Gates: `make lexer && make test && make verify-core` before PR.
- Grammar tripwire must stay 31/0; parser.y changes require the goo-grammar
  skill and its conflict-ledger procedure.
- Float adaptation machinery lives in src/types/expression_checker.c
  (`adapt_untyped_float_operand`, `check_conversion_operand_range`,
  `is_untyped_float_rooted`) — extend it; don't build parallel folders.
